先拉一下image:

```shell
docker pull ubuntu:22.04
```

然后随便创建个目录，再在目录里创建dockerfile

使用以下dockerfile建立容器：
```shell
FROM ubuntu:22.04
WORKDIR /root
# noninteractive 配置时不需要输入任何信息
ENV DEBIAN_FRONTEND="noninteractive"
# 配置时区
ENV TZ=Asia/Shanghai
# 配置apt源
RUN sed -i 's@//.*archive.ubuntu.com@//mirrors.ustc.edu.cn@g' /etc/apt/sources.list
# 安装必要的软件
RUN apt update && apt install -y init locales
# 配置语言环境
RUN locale-gen en_US.UTF-8
# 配置 entrypoint 为 init, 是 init 可以作为 pid 1 运行, 并且可以启动 systemd
RUN passwd -d root
ENTRYPOINT ["/usr/sbin/init"]
```

使用以下命令创建镜像与容器：

```shell
docker build -t ubuntu:22.04-systemd .
docker run -p 8080:80 -it --privileged --name ubuntu -h ubuntu ubuntu:22.04-systemd
```

运行容器进入ubuntu后，login时输入用户名root，要求输入密码不用管，直接回车。

进入容器后，执行下列命令：

```shell
apt update
apt install nginx
rm /etc/nginx/sites-enabled/default
vim /etc/nginx/sites-enabled/test.conf
```

在`test.conf`中粘贴以下内容:

```shell
server {
    listen 80;
    root   /var/www/html;
    server_name  _;
    #charset koi8-r;
	access_log /var/log/nginx/access.log;
    location / {
        index index.php index.html index.htm;
    }
    #error_page  404              /404.html;
    #redirect server error pages to the static page /50x.html
    #
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /var/www/html;
    }
    #pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
    #
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;
    }
}
```

然后保存退出，后执行下列命令：

```shell
apt install mariadb-server mariadb-client
systemctl start mariadb
systemctl enable mariadb
mysql
```

执行下列sql语句：

```sql

create database db_discuz default character set utf8 collate utf8_general_ci;

```

然后exit退出mariaDB

然后开始安装php

```shell
apt install software-properties-common
add-apt-repository ppa:ondrej/php
add-apt-repository ppa:apt-fast/stable
apt update
apt install apt-fast
apt-fast install php7.4 php7.4-bcmath php7.4-cli php7.4-common php7.4-curl php7.4-fpm php7.4-gd php7.4-gmp php7.4-imagick php7.4-intl php7.4-json php7.4-mbstring php7.4-mysql php-pear php7.4-xml php7.4-xmlrpc php7.4-zip php7.4-imap -y
```

然后向`/var/www/html/index.php`中写入内容：


### day2 搭建API靶场、API安全

crAPI是OWASP推出的一个API安全靶场，其模拟一个汽车维修管理平台，在API设计上故意暴露了许多漏洞。

#### 认识API安全

API安全有以下几个特性与其对应风险

- 机密性——明文传输、token泄露
- 完整性——request/response被中间人拦截并修改
- 可用性——DDOS等攻击
- 认证——弱口令、token伪造
- 授权——越权（水平/垂直越权）
- 审计——无日志、日志被删

#### 常见的API安全漏洞及其攻击手法

##### 失效的对象级授权 Broken Object Level Authorization (BOLA)

基本原理：服务端读取数据时，只按照读取对象的id读取数据，但是并未验证对象是否属于该用户，从而造成攻击者可以非法读取其他用户的隐私数据

对应crapi challenge 1 与 challenge 2

在challenge1中，使用yakit抓包，可以获取到泄露的vehicle id

使用该vehicle id可以未授权直接访问该车辆的信息，而不会检验车主

在challenge2中，可以获取到report地址

直接访问会提示JWT token，导致401，但在yakit中带上我们自己的token重放也还是能越权访问到其他车辆信息，说明其后端只是要求你发送token，但没有对token与车辆id是否合法进行校验。

##### 失效的用户身份验证

基本原理：在身份验证API处，对OTP、token等未限制爆破，

其他用户邮箱可从api返回信息进行泄露。得到邮箱后在忘记密码处，在v2下对他人验证码进行爆破。

##### 过量数据暴露

基本原理：API返回信息非最小满足可用，而是返回了过量的（可能很多根本用不到）信息，这些信息可以被攻击者所获取

3. union 联合注入

此时要开始注意查询回显位，因为union返回多行，后端却只取第一行，此时需要让第一行不存在⇒使得第一行查询不到任何数据，此时后面的数据就会上位，从而方便我们获得想要的数据

`http://127.0.0.1:8888/Less-1/?id=-1' union select 1,2,3--+`

这里注意要使得id为一个不存在的数字，其原理是：sql查询可能返回多条记录，但前端会选择第一条记录的其中某些字段显示，而union联合查询会将后续的查询结果跟在前一个查询结果的后面，我们要做的就是使得前面的查询查不到任何记录，此时后面我们做的查询就会顶替到第一行。

##### less 5

这一关中，前端不再返回多余数据库内容，但是会返回报错信息。我们就需要用到报错注入。

报错注入的原理在于，让回显的报错信息中夹杂我们想要的信息

报错注入可以基于的函数多种多样，这里只介绍其中用得比较多的几种：

**`extractValue()`报错注入**

`extractValue()`函数用来查询xml里面的内容；该函数包含两个参数：
- arg1：XML文档对象名称;
- arg2：路径

```sql
extractvalue('目标xml文件名=>列名','在xml中查询的字符串');
select extractValue(doc, '/book/author/suername') from xml;
```

假设有一个包含XML数据的表`books`，其中一列为`book_info`包含以下内容：

```xml
<book>
	<title>MySql Cookbook</title>
	<author>Paul DuBois</author>
	<year>2014</year>
</book>
```

我们可以使用extractvalue()来提取特定节点下的值。例如，要获取书籍标题(title)：

```sql
select extractvalue(book_info, '/book/title') from books;
```

返回的结果为:

```sql
+-------------------+
| title             |
+-------------------+
| MySQL Cookbook    |
+-------------------+
```

第一个参数可以随便写，无所谓，重点在于第二个参数

如果我们输入的Xpath_string不对就会报错，而如果页面回显sql报错信息，就可以得到我们想要的信息了

```sql
select extractvalue(book_info, '/book/titles') from books;
--+ title多写了一个s，并不会报错，只是查询不到信息
select extractvalue(book_info, '~book/title') from books;
--+ 把查询参数格式符号写错，会报错
select extractvalue(1, concat(0x7e, (select database()))) from books;
--+ 会先把select database()执行，再与0x7e拼接，发现拼接后的语句有错，此时报错会将select database()的信息一并暴露
--+ concat() 字符串拼接 0x7e就是飘号~(ascii)
```

基于以上原理，可以爆库名：

`http://127.0.0.1:8888/Less-5?id=1' and 1=extractvalue(1, concat(0x7e, (select database())))--+`
